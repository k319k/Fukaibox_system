C. ダークモード & カスタムバリアント

Dark Mode: class ストラテジー。bg-background は #1a1110 を固定。

Glassmorphism: 浮遊要素には backdrop-blur-md と bg-opacity-70 を適用。

2. コンポーネント別・文字色詳細仕様

2.1 Alert (警告・通知)

Alert の文字色は、背景色に合わせて厳格に指定すること。

Warning (警告):

variant="flat" color="warning"

base: bg-[#fbe7a6] (Tertiary Container)

mainWrapper: text-[#564419] (On Tertiary Container)

Danger (危険/エラー):

variant="flat" color="danger"

base: bg-[#ffdad6] (Error Container)

mainWrapper: text-[#93000a] (On Error Container)

2.2 Button (アクション)

最優先: color="primary" className="text-white"

標準 (Tonal): variant="flat" color="primary" className="bg-[#ffdad5] text-[#73342b]"

2.3 Input & Textarea (入力)

エラー状態:

isInvalid={true} 時の errorMessage は必ず #ba1a1a (Error) を使用。

背景は bg-[#ffdad6] への切り替えを許容。

3. 「封解Box」固有の高度なUIパターン

3.1 調理進捗・推敲ボード (Cooking/Drafting)

推敲提案中: セクション全体を bg-[#fbe7a6] (Warning Container) で包み、文字色を #564419 に固定して「検討が必要」であることを示す。

3.2 儀員ランキング

上位ランク: bg-[#ffdad5] (Primary Container) を使用し、文字色を #73342b で表示。

4. 品質保証チェックリスト (Final Guardrails)

❌ 純白 (#FFFFFF) 禁止: 背景は必ず #fff8f6 (background) を使用。

❌ 原色赤 (#FF0000) 禁止: エラーは必ず #ba1a1a または上記の On-Error カラーを使用。

❌ コントラスト不足禁止: 薄い色の背景に白文字を載せることを厳禁とする。必ず定義された On-Container カラーを使用せよ。

✅ マイクロインタラクション: 全ての Button に whileTap={{ scale: 0.95 }} を付与せよ。
